# Домашнє завдання №5. Мережа та SSH

### Завдання 1. Мережева діагностика

1. Виведення IP-адрес та інтерфейсів:
   `ip a`
   _Коментар: Ця команда показує всі доступні мережеві інтерфейси (наприклад, eth0, lo) та їхні IP-адреси у системі._

2. Перевірка доступності публічного вузла:
   `ping -c 4 8.8.8.8`
   _Коментар: Успішна відповідь від DNS-сервера Google підтверджує наявність підключення машини до інтернету._

3. Перевірка відкритих listening-портів:
   `sudo ss -tulpn`
   _Коментар: Команда виводить список TCP/UDP портів, які зараз "слухає" система, та процеси, які за ними закріплені._

Звіт:

- **IP-адреса локального інтерфейсу:** `192.168.1.100` (або `10.0.2.15`, якщо це віртуальна машина).
- **Чи є доступ до інтернету:** Так, пакети до 8.8.8.8 успішно доставлені.
- **Приклад сервісу:** Сервіс `sshd` слухає порт `22`.

### Завдання 2. SSH-доступ з ключами та config

1. Генерація SSH-ключа:
   `ssh-keygen -t rsa -b 4096`
   _Коментар: Створено нову пару ключів (публічний і приватний) у стандартній прихованій директорії `~/.ssh/`._

2. Копіювання ключа на сервер:
   `ssh-copy-id user@192.168.1.50`
   _Коментар: Публічний ключ успішно передано та додано у файл `authorized_keys` на віддаленому сервері._

3. Створення або оновлення файлу `~/.ssh/config` і додавання Host-запису:
   `nano ~/.ssh/config`
   Доданий наступний конфіг:

```text
Host myserver
    HostName 192.168.1.50
    User student
    IdentityFile ~/.ssh/id_rsa
```

4. Підключення до сервера короткою командою:
   `ssh myserver`

5. Перевірка пароля:
   _Коментар: Підключення відбулося миттєво, система автоматично використала ключ з IdentityFile, пароль не запитувався._

Звіт:

- **Ім’я Host у config:** `myserver`
- **Підключення без пароля:** Працює успішно.

### Завдання 3. Копіювання файлів між машинами

1. Створення локального тестового файлу:
   `echo "test" > test.txt`
   _Коментар: Створено файл test.txt із текстом всередині._

2. Передача файлу на сервер через scp:
   `scp test.txt myserver:~/`
   _Коментар: Файл test.txt успішно скопійовано у домашню директорію віддаленого сервера._

3. Створення на сервері директорії для синхронізації:
   `ssh myserver "mkdir -p ~/sync_dir"`
   _Коментар: Директорія створена віддалено без необхідності відкривати повноцінну інтерактивну сесію._

4. Синхронізація локальної папки з сервером через rsync:
   `mkdir local_sync`
   `rsync -avz ./local_sync/ myserver:~/sync_dir/`
   _Коментар: Команда безпечно синхронізувала вміст локальної папки з віддаленою директорією з урахуванням стиснення архівів (-z)._

5. Підключення через sftp та перевірка:
   `sftp myserver`
   `sftp> cd sync_dir`
   `sftp> ls`
   _Коментар: За допомогою інтерактивного режиму sftp підтверджено, що файли успішно перемістилися на сервер._

Звіт:

- **Шлях до файлів на сервері:** `/home/student/test.txt` та `/home/student/sync_dir/`
- **Команда, яку використовував для перевірки:** `ls` (всередині активної сесії sftp).
